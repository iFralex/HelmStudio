# Plan: Project Bootstrap

**Branch:** `feat/01-project-bootstrap`
**Wave:** 1
**Depends on:** —
**Estimated effort:** 1–2 days

## Overview

Initialise the Next.js 15 project with TypeScript strict mode, Tailwind v4, shadcn/ui, ESLint, Prettier, Vitest, Playwright, and the single-password authentication described in spec §11.7. Lays the entire UI shell that every later plan extends; pages from later plans plug into the protected layout established here. No business data is rendered yet.

## Context

The system is single-operator (spec §2 Non-Goals), so authentication is a single shared password held in `.env` compared in a middleware. Sessions are signed cookies, not a `users` table. All app routes live under a route group `(app)` that requires auth; `/login` is the only public route. UI strings are Italian per project policy; everything else (identifiers, log messages, comments) is English.

## Validation Commands

- `pnpm install`
- `pnpm typecheck`
- `pnpm lint`
- `pnpm format:check`
- `pnpm dev` → http://localhost:3000 redirects to `/login`, log in with `ADMIN_PASSWORD`, see empty dashboard

```typescript
export function verifyPassword(plain: string): boolean;
// timing-safe compare against process.env.ADMIN_PASSWORD

export function createSessionCookie(): { value: string; expires: Date };
// HMAC-SHA256({ exp: Date.now() + 30d }, SESSION_SECRET) — base64url-encoded

export function verifySessionCookie(token: string | undefined): boolean;
// validates HMAC and `exp` field
```
